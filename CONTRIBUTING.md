# 貢献ガイドライン

本リポジトリへの貢献をありがとうございます！本マニュアルの改善にご協力いただけることを歓迎します。

## 貢献の方法

### 問題の報告

バグや誤りを発見した場合、または改善提案がある場合は、GitHubのIssuesで報告してください。

**Issueを作成する際は、以下を含めてください：**
- 問題の説明
- 再現手順（該当する場合）
- 期待される動作
- 実際の動作
- 環境情報（PHITSのバージョン、OSなど）

### プルリクエスト

修正や改善を提案する場合は、Pull Requestを作成してください。

**プルリクエストを作成する前に：**
1. 既存のIssueを確認し、重複していないか確認してください
2. 大きな変更の場合は、事前にIssueで提案を議論してください

**プルリクエストの作成手順：**
1. リポジトリをフォーク
2. 新しいブランチを作成（`git checkout -b feature/amazing-feature`）
3. 変更をコミット（`git commit -m 'Add some amazing feature'`）
4. ブランチにプッシュ（`git push origin feature/amazing-feature`）
5. Pull Requestを作成

## コーディング規約

### マークダウンの記述

- 見出しは適切な階層で使用してください
- コードブロックには適切な言語タグを付けてください
- リンクは明確で説明的なテキストを使用してください

### PHITS入力ファイルの記述

- コメントを適切に使用してください
- パラメータの説明を追加してください
- 実際に動作する入力ファイル例を提供してください

### ドキュメントの構造

- 新しいセクションを追加する場合は、既存の構造に従ってください
- セクション番号は適切に調整してください
- 目次（README.mdの冒頭）を更新してください（該当する場合）

## レビュープロセス

1. すべてのプルリクエストはレビューされます
2. フィードバックがあった場合は、対応をお願いします
3. 承認後、変更がマージされます

## 行動規範

- 建設的で尊重のあるコミュニケーションを心がけてください
- 多様な意見や経験を歓迎します
- 不適切な行動は容認されません

## 質問

質問や不明な点がある場合は、Issueで質問してください。
